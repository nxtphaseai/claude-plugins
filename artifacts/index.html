nxtphaseai

NxtPhase plugin marketplace

A small, opinionated set of plugins for Claude Code. Subscribe once, install whatever fits, get updates by re-subscribing.

Subscribe

# one-time, register the marketplace
claude /plugin marketplace add nxtphaseai/claude-plugins

# browse and install
claude /plugin install agent-eval@nxtphaseai
claude /plugin install ask-visual@nxtphaseai

# later, pull updates
claude /plugin marketplace update nxtphaseai

Plugins

agent-eval

v0.3.0 evaluation

Auto-grades the agent's last turn against your prompt, and against optional project rules in eval/eval.md, then appends a red/green scored card to eval/eval.html. Wires deterministic linters and typecheckers into the same grading pass as the LLM rubric, so you see in one place whether the agent shipped what you asked for and whether it broke any project rules along the way.

ask-visual

v0.1.0 interactive

Visual AskUserQuestion. Instead of an ASCII menu, the agent writes a real HTML form (cards, sliders, color pickers, anything the browser supports), the plugin spins up a one-shot localhost web server, the user picks in their browser, and the answer flows back as JSON. No persistent backend, server dies on submit.

For authors

Add a new plugin in three steps:

  1. Create plugins/<name>/ with at minimum a .claude-plugin/plugin.json and a README.md. Add commands/, hooks/, agents/, bin/ as needed.
  2. Register it in .claude-plugin/marketplace.json and bump metadata.version.
  3. Open a PR to main. Once merged, teammates pick it up via claude /plugin marketplace update nxtphaseai.

Conventions: hook commands use ${CLAUDE_PLUGIN_ROOT} so they survive both install paths. Hooks fail soft (missing tools log to stderr and exit 0, never block the user's turn). Each plugin ships its own install.sh for environments without /plugin support.
