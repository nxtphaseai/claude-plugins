Pick a tween easing for the shape

A live demo of the choices the agent serves through ask-visual. Each card runs the animation on loop so the personality of the curve is visible, not just labeled.

cubic-bezier(.42,0,.58,1)

Ease-in-and-out

Smooth, neutral, invisible

Speeds up, peaks at the middle, slows down. The motion of someone walking through a doorway, measured, polite, almost forgettable. Use when you want the animation itself to disappear and only the result to register.

spring( stiffness, damping )

Spring

Bouncy, physical, alive

Pulled past the target by inertia, snaps back, settles after a few oscillations. The motion of a screen door slamming shut behind you. Use when the UI should feel like it has weight, like a real object with mass and friction.

cubic-bezier(.68,-0.6,.32,1.6)

Anticipate

Wind-up, dramatic, cinematic

Pulls back first, then lunges past. The crouch before a jump, the wind-up before a punch. Use when you want the user's eye to land on the motion before it happens, priming attention and announcing importance.

How this card was served

The agent wrote the body of this exact form to a temp file and ran:

python3 .claude/bin/ask-visual.py /tmp/ask.html

A one-shot HTTP server bound a free localhost port, opened the URL in the user's default browser, blocked until the form was submitted, then printed the form data as JSON to stdout and exited. No persistent backend. The agent read the JSON and continued the conversation.
