ask-visual

Visual AskUserQuestion. The agent writes a real HTML form, the user picks/types/drags/clicks in their browser, the answer flows back as JSON.

v0.1.0 Python stdlib only ~140 lines MIT

Instead of an ASCII menu or a one-line text question, the agent writes a real HTML form with cards, sliders, color pickers, drag-to-reorder, anything the browser supports. The plugin spins up a tiny one-shot localhost web server, opens it in the user's default browser, the user picks/types/drags/clicks, the form is POSTed back, the script prints the result as JSON to stdout, and the server exits. No daemon, no cleanup.

Install

Through the marketplace:

claude /plugin marketplace add nxtphaseai/claude-plugins
claude /plugin install ask-visual@nxtphaseai

Or manually, into the current project:

git clone <this-repo>
bash plugins/ask-visual/install.sh           # into ./.claude/ of the current git repo
bash plugins/ask-visual/install.sh --user    # user-wide into ~/.claude/

Restart Claude Code so the slash command loads. To remove:

bash plugins/ask-visual/install.sh --uninstall

What it installs

PathRole
<scope>/commands/ask-visual.mdThe /ask-visual slash command, the instructions the agent reads.
<scope>/bin/ask-visual.pyThe HTTP server / form runner. Python stdlib only.

How the agent uses it

The agent writes the body of a form to a temp file (no <html>, <form>, or submit JS, those are wrapped on for it), then runs:

python3 .claude/bin/ask-visual.py /tmp/ask.html

The script:

  1. Binds a random free port on 127.0.0.1.
  2. Opens that URL in the user's default browser.
  3. Blocks until a POST /submit arrives.
  4. Prints the form data as a single JSON object to stdout.
  5. Exits.

Inside the form, every name-bearing input is captured. The clicked button's name and value are included in the submission, so a card layout with one <button name="choice" value="card-a"> per card works out of the box, clicking the button picks the card and submits the form in one motion. Multi-select dropdowns return arrays.

Example

The agent wants to ask: "pick a card design and an accent color". It writes this to /tmp/ask.html:

<h1>Pick a card design and accent color</h1>
<div style="display:grid;grid-template-columns:repeat(3,1fr);gap:14px">
  <div style="border:1px solid #ccc;border-radius:8px;padding:14px">
    <h3>Minimal</h3>
    <button name="card" value="minimal">Pick this</button>
  </div>
  <div style="border:1px solid #ccc;border-radius:8px;padding:14px;
              box-shadow:0 2px 8px rgba(0,0,0,.08)">
    <h3>Lifted</h3>
    <button name="card" value="lifted">Pick this</button>
  </div>
  <div style="border:2px solid #1a1a1a;border-radius:8px;padding:14px">
    <h3>Bold</h3>
    <button name="card" value="bold">Pick this</button>
  </div>
</div>
<p><label>Accent color:
  <input type="color" name="accent" value="#1a1a1a"></label></p>

The user clicks the "Lifted" card's button after picking a blue accent. stdout receives:

{"accent":"#2c5aa0","card":"lifted"}

The agent reads that and continues the task with the user's choice in hand. A live demo of this same pattern lives in demo-easings.html.

Limits and behavior

  • Localhost only. Binds 127.0.0.1, no exposure to the network.
  • One submission, then exit. The form gets disabled after submit so refresh or double-click cannot double-fire.
  • Default 10-minute timeout waiting for submission. Override via ASK_VISUAL_TIMEOUT=<seconds>. On timeout the script exits 3.
  • No persistent backend. No PIDs to track, no temp dirs to clean up, nothing left running between turns.

When NOT to use it

Headless or SSH-only environments, webbrowser.open no-ops silently there. Fall back to AskUserQuestion.
Yes/no or one-of-two text questions, AskUserQuestion is faster.
Anything where the user types a paragraph, text Q&A handles that fine.

What it does NOT do

  • Doesn't modify your code.
  • Doesn't store anything between runs, the server is one-shot.
  • Doesn't talk to any third party. Localhost only.
  • Doesn't authenticate, assumes single-user local environment.
